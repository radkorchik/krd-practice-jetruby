# Выполненное задание по лекции

### 1) Что такое debugging?
Debugging — процесс поиска, анализа и устранения ошибок (bugs) в ПО.

### 2) Что такое exception (исключение)? Из чего состоит?
Exception — событие времени выполнения, сигнализирующее об ошибке.
Обычно включает:

- **message** (текст ошибки),
- **тип исключения** (например `NoMethodError`, `ArgumentError`),
- **stack trace** (где и по какому пути возникла ошибка).

### 3) Что такое stack trace?
Stack trace — стек вызовов: список файлов, классов, методов и строк кода, через которые программа дошла до ошибки.
Нужен для точного определения источника проблемы.

### 4) Что такое logs/логирование? Какие уровни логирования есть и чем отличаются?
Логи — упорядоченные по времени записи о событиях приложения (запросы, SQL, ошибки, служебные сообщения).

Типовые уровни:

- `debug` — самый подробный, для отладки;
- `info` — информационные сообщения о штатной работе;
- `warn` — предупреждения;
- `error` — ошибки, но сервис продолжает работать;
- `fatal` — критические ошибки, возможно завершение процесса.

### 5) Какие инструменты для дебага предоставляет Ruby on Rails?

- Exception pages (в development);
- stack trace;
- Rails logger (`Rails.logger`);
- `rails console` / `rails console --sandbox`;
- встроенный debugger (`debug` gem);
- helper-методы в представлениях (`debug`, `inspect`);
- (опционально) `pry-rails`, `pry-byebug`.
